# Novel Writer Systematic Creative Workflow

## When the user says "I want to write a novel"

**Immediately activate this workflow guide.**

## Seven-Step Methodology Overview

```
1. Constitution  → Establish creative principles
2. Specify       → Define story specifications
3. Clarify       → Clarify ambiguities
4. Plan          → Create creative plan
5. Tasks         → Break down execution tasks
6. Write         → AI-assisted writing (with pre-write checklist)
7. Analyze       → Quality verification and analysis
```

## Detailed Step Guidance

### Step 1: Establish Creative Principles

> "Let's start by defining your creative principles using the Constitution step. This establishes your core values, quality standards, and the non-negotiable rules for this story."

**Why this matters**: Acts as your "creative constitution" — all future decisions will reference these principles to maintain consistency.

**What to include**:
- Core values (the theme or message you want to convey)
- Quality baseline (absolute non-negotiables)
- Style principles (language, pacing, atmosphere)
- Content principles (character, plot, and worldbuilding norms)

**Typical Duration**: 15-20 minutes

### Step 2: Define Story Specifications

> "Now let's create a story specification document, much like a product spec. Think like a product manager: what is this story, who is it for, and what makes it unique?"

**What to include**:
- Logline (one-sentence summary)
- Target audience
- Core conflict
- Main characters
- Success criteria

**Why this matters**:
- Clarifies "who you're writing for" and "what you're writing"
- Sets measurable success criteria
- Provides clear targets for subsequent creation

**Typical Duration**: 30-45 minutes

**Special Markers**:
- `[Needs Clarification]` - Marks points that need further definition
- `[Core Requirement]` - Non-negotiable requirements
- `[Optional Feature]` - Nice-to-have content

### Step 3: Clarify Key Decisions

> "Let's resolve any unclear aspects by answering up to 5 targeted questions. The AI will identify ambiguities in your specification and help you make clear decisions."

**Why this matters**: Ambiguity in planning equals chaos in writing.

**Process**:
- AI identifies ambiguities in the specification
- Generates up to 5 critical questions
- Interactive Q&A, answers directly update the specification
- Preserves decision history for backtracking

**Typical Questions**:
- "Is the protagonist's core motivation revenge or justice?"
- "Is the pacing fast and action-packed, or a slow-burn?"
- "Is the ending a Happy Ever After (HEA) or a tragedy?"

**Typical Duration**: 10-15 minutes

### Step 4: Create Creative Plan

> "Let's create the chapter structure and technical approach. This is where you decide HOW to implement your specification."

**What to design**:
- Chapter breakdown and active plotlines
- Pacing and tension distribution
- Foreshadowing plan
- Character arc mapping

**Why this matters**:
- Turns abstract requirements into concrete solutions
- Selects appropriate writing methods and structures
- Designs the specific implementation of plot, characters, and worldbuilding

**Technical Decision Examples**:
- "Use a seven-point story structure to enhance suspense"
- "Adopt a multi-POV narrative to show complexity"
- "Increase mystery through a limited perspective"

**Typical Duration**: 45-60 minutes

### Step 5: Break Down Execution Tasks

> "Let's generate an actionable task list with priorities, dependencies, and estimated effort."

**Task Types**:
- Chapter writing tasks
- Character profile refinement
- Worldbuilding expansion
- Revision and polishing tasks

**Task Markers**:
- `[P]` - Can be executed in parallel
- `[Dep:X]` - Depends on the completion of task X
- `[High Priority]` - High priority task

**Why this matters**:
- Breaks massive goals into small steps, lowering the barrier to entry
- Clarifies priorities and dependencies
- Supports parallel creation to improve efficiency

**Typical Duration**: 20-30 minutes

### Step 6: Start Writing

> "Now we write chapter by chapter with AI assistance. The specification and plan guide every chapter, ensuring consistency through the **Pre-Write Checklist**."

**During Writing**:
- Genre knowledge skills activate automatically based on context
- Consistency checker runs in the background
- Pacing monitor alerts when issues are detected
- **CRITICAL: The Pre-Write Checklist is executed before every chapter to load necessary context.**

**Execution Principles**:
- Strictly follow the principles in the Constitution
- Reference the technical solutions in the Plan
- Complete specific items from the Tasks
- Maintain consistency with the Specification

**Recommended Pace**:
- 1-2 chapters per writing session
- Stop and review every 3-5 chapters
- Maintain forward momentum; avoid over-editing during the first draft

### Step 7: Quality Check (Analyze)

> "Run an analysis periodically to maintain quality and catch issues early."

**Two Analysis Modes**:
- **Framework Analysis** (Pre-writing): Validates the planning
- **Content Analysis** (Post-writing): Validates the execution

**Verification Dimensions**:
- Constitution compliance (does it violate core principles?)
- Specification fulfillment (did it achieve the requirements?)
- Content consistency (plot, timeline, characters)
- Quality standards (does it meet the set baseline?)

**When to Run**:
- After completing the first 3 chapters (early validation)
- Every 5 chapters (regular checkup)
- After completing the draft (comprehensive review)

## When Users Deviate

If a user tries to skip steps or jump straight to writing without a plan:

**Gentle Reminder**:
> "I notice you're jumping straight into writing. The strength of this system lies in systematic upfront planning. Would you like to establish a solid foundation first? It makes the writing process much smoother and helps maintain consistency."

**If the user insists on skipping**:
> "I understand — you can always create the planning documents later. However, please note that without the specification and plan, you'll need to track consistency manually."

## Adjustments for Different Project Sizes

### Short Story (10k-30k words)
**Simplified Flow**: `Specify → Clarify → Write → Analyze`
**Focus**: Clear core conflict and turning points.

### Novella / Standard Novel (50k-100k words)
**Standard Flow**: Full Seven Steps.
**Focus**: Planning and task breakdown.

### Long Web Serial (200k+ words)
**Enhanced Flow**: Repeat the cycle `Plan → Tasks → Write → Analyze` for each major arc.
**Focus**: Continuous consistency verification.

## Handling Common Questions

### "I already have an outline."
> "Great! Let's convert it into our format. This makes it machine-readable and enables all tracking features."
**Process**: Import existing outline → Format it → Fill gaps → Create technical plan.

### "This looks like a lot of setup."
> "Setup takes 30-60 minutes but saves hours of revision. You're essentially creating the 'source code' for your novel. Without it, changes require manual full-text rewrites."

### "Can I just use parts of the workflow?"
> "Yes, but some steps are interdependent. Writing needs the specification and plan to work effectively. Analysis needs written chapters. The minimum viable workflow is: Constitution → Specify → Write."
